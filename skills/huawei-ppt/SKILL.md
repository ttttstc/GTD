# Huawei PPT — 华为风格演示文稿

> 本 skill 继承 ppt-forge 流水线框架，品牌预设使用华为视觉规范。

## 华为品牌预设

| 参数 | 华为预设值 |
|------|---------|
| 主色 | #C7020E（华为红）、#1A1A1A（黑）、#F5F5F5（浅灰） |
| 圆角 | 0px（直角体系） |
| 密度 | 高密度（模块化 Boxy） |
| 字重 | 粗体标语 + 细体数据 |
| 图标 | 面性/实心 |
| 留白 | 最小化 |
| 字体 | 华为体系：无衬线 geometric，中文用 HarmonyOS Sans / MiSans |

详细规范见 `refs/huawei-brand.md`

## 典型页型

| 页型 | 说明 | 典型场景 |
|------|------|---------|
| 战略总览 | 高密度多模块，展示全局视图 | 战略分析、业务规划 |
| 汇报页 | 数据驱动，KPI + 趋势 | 季度汇报、月度总结 |
| 架构页 | 模块化分层，展示系统结构 | 技术架构、方案设计 |
| 对比分析 | 双列/多列对比，展示选项 | 方案对比、选型分析 |
| 发布会结论 | 简洁有力，单点突出 | 产品发布、对外演讲 |

详细模板见 `refs/huawei-archetypes.md`

## 密度标准

华为 PPT 的核心特征是**高信息密度**。不同页型有不同的密度上限：

| 页型 | whitespace 上限 | 模块数下限 | 字号下限 |
|------|----------------|-----------|---------|
| 战略总览 | 25% | 6 | 8px |
| 汇报页 | 30% | 4 | 9px |
| 架构页 | 20% | 8 | 8px |
| 对比分析 | 25% | 4 | 9px |
| 发布会结论 | 40% | 2 | 12px |

详细标准见 `refs/huawei-density-rules.md`

## 使用流程

```
1. 继承 ppt-forge 的开局参数确认
   → 锁定：archetype / 华为品牌 / 受众 / 场景 / 观看模式

2. S 场景：风格定调
   → 使用 huawei-brand.md 的 CSS 变量制作核心页

3. A 场景：Slide 制作
   → 使用 huawei-archetypes.md 的模板
   → 遵循 huawei-density-rules.md 的密度标准

4. R 场景：视觉审查
   → 品牌审查时挂载华为审美滤镜

5. E + D：导出 + 交付
   → 同 ppt-forge 标准流程
```

## 华为 PPT 特征速查

### 视觉特征
- 红色作为强调色，大量使用（模块标题、分隔线、CTA）
- 黑色用于主标题和重要文字
- 灰色用于次要信息和背景色块
- 直角边框，无圆角或极小圆角

### 内容特征
- 数据驱动：每个结论都有数据支撑
- 模块化：内容切分为独立模块，清晰分区
- 层级清晰：L1 标题 → L2 副标题 → L3 正文 → L4 辅助
- 密度优先：信息量最大化，减少留白

### 禁忌
- ❌ 大量留白（华为不是 Apple）
- ❌ 圆角卡片（华为是直角体系）
- ❌ 轻柔配色（华为需要严肃商务感）
- ❌ 单页单信息（华为是高密度多模块）
