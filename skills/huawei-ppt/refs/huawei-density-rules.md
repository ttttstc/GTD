# 华为密度标准

> 华为 PPT 的信息密度规范。

## 核心理念

**华为 PPT 不是 Apple。华为是信息密集型风格，追求"一页一世界"。**

- Apple：每页一件事，大量留白
- 华为：每页多模块，信息密度最大化
- 通用：视受众和场景而定

## 密度等级

| 等级 | whitespace | 模块数 | 适用页型 |
|------|-----------|--------|---------|
| 极高 | < 20% | 8-12 | 战略总览、架构页 |
| 高 | 20-30% | 6-8 | 数据洞察、多维分析 |
| 中 | 30-40% | 4-6 | 汇报页、对比分析 |
| 低 | > 40% | 2-4 | 发布会结论、封面 |

## 密度指标

### whitespace 计算

```
whitespace% = (空白像素) / (总像素) × 100%
```

### 各页型密度上限

| 页型 | whitespace 上限 | element count 下限 | text nodes 下限 |
|------|----------------|-------------------|----------------|
| 战略总览 | 25% | 15 | 40 |
| 汇报页 | 30% | 10 | 30 |
| 架构页 | 20% | 12 | 35 |
| 对比分析 | 25% | 8 | 25 |
| 发布会结论 | 40% | 4 | 15 |

### 华为 vs Apple vs 通用

| 指标 | 华为 | Apple | 通用 |
|------|------|-------|------|
| whitespace | 20-30% | 50-60% | 30-40% |
| 模块数 | 6-12 | 1-3 | 3-6 |
| 字号下限 | 8px | 14px | 10px |
| 信息量 | 极高 | 极低 | 中等 |

## 填充手段使用规则

华为 PPT 必须使用多种填充手段，避免单调：

| 填充手段 | 华为推荐度 | 使用建议 |
|---------|-----------|---------|
| KPI 数字块 | ★★★★★ | 必用，每页至少 2 个 |
| SmartArt 流程图 | ★★★★★ | 必用，展示流程 |
| 数据表格 | ★★★★☆ | 推荐，表格密度高 |
| 截图/图片 | ★★★★☆ | 推荐，真实素材 |
| 色块分区 | ★★★☆☆ | 适度，分割区域用 |
| 图标+文字 | ★★★☆☆ | 适度，不要滥用 |
| 多级字号 | ★★★★★ | 必用，4 层以上 |

**最少 3 种填充手段**：KPI + 流程图 + 表格/图片

## 模块化 Boxy 布局

华为风格的核心是"模块化 Boxy"：

```
┌─────────────────────────────────────────────────────┐
│  ┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐          │
│  │ Box 1 │ │ Box 2 │ │ Box 3 │ │ Box 4 │          │
│  └───────┘ └───────┘ └───────┘ └───────┘          │
│  ┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐          │
│  │ Box 5 │ │ Box 6 │ │ Box 7 │ │ Box 8 │          │
│  └───────┘ └───────┘ └───────┘ └───────┘          │
└─────────────────────────────────────────────────────┘
```

### 模块布局规则

- **grid-template-columns**: repeat(4, 1fr) 或 repeat(3, 1fr)
- **gap**: 4px（紧凑）
- **border**: 1px solid #E0E0E0（卡片边框）
- **padding**: 5-8px
- **radius**: 0px（直角）

## 密度检查工具

### Pre-flight 密度检查

```javascript
// 密度计算脚本
const calculateDensity = (container) => {
  const totalArea = container.offsetWidth * container.offsetHeight;
  const whitespace = calculateWhitespace(container);
  const elementCount = container.querySelectorAll('*').length;
  const textNodes = countTextNodes(container);

  return {
    whitespacePct: (whitespace / totalArea * 100).toFixed(1),
    elementCount,
    textNodes,
    density: (100 - whitespacePct).toFixed(1)
  };
};

// 检查标准（华为高密度）
const huaweiHighDensity = {
  whitespaceMax: 30,  // 30%
  elementMin: 10,
  textNodesMin: 30
};
```

## 常见密度问题

| 问题 | 原因 | 修复 |
|------|------|------|
| 密度太低（whitespace > 40%） | 内容太少 | 加 KPI、加模块、加图表 |
| 信息挤在一起 | 字号太小 | 按页型下限调整字号 |
| 模块数太少 | 缺乏内容分层 | 拆分内容为多模块 |
| 表格太空 | 数据不够 | 加更多行/列或加对比 |

## 密度调优流程

1. **制作完成后跑密度计算**
2. **如果 whitespace > 上限**：加内容（KPI、模块、流程图）
3. **如果 element count < 下限**：加填充手段
4. **如果字号偏小**：检查 viewing mode，presentation 可适当缩小
5. **完成后截图确认视觉**
